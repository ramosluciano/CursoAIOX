## 📋 Rollback Procedure

**If critical issue detected in production:**

1. **Detection:** Error spike in monitoring or user reports
2. **Assessment:** Severity classification (critical/high/medium/low)
3. **Decision:** Rollback vs. hotfix (critical → rollback, high/medium → assess)
4. **Execution:**
   ```bash
   git revert HEAD~1  # Revert last commit
   git push origin main  # Push revert
   npm run build  # Verify build succeeds
   # Deploy reverted version to production
   ```
5. **Communication:** Notify team of rollback and ETA for fix
6. **Root Cause:** Post-mortem and prevention strategies
